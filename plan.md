# 服务部署架构师演练场 (Deployment Playground) 项目计划

## 1. 项目愿景
打造一个面向初级架构师的交互式练习平台，通过可视化的拖拽操作，模拟从单体应用到微服务集群的部署演进过程。用户可以在安全的沙盒环境中体验流量冲击、服务故障、扩容缩容等真实场景，学习架构设计原则。

## 2. 核心功能模块

### 2.1 可视化架构设计器 (Visual Designer)
- **组件库 (Palette)**: 提供标准化的基础设施组件。
  - *计算与编排*: 物理机, 虚拟机, 容器 (Pod), Auto Scaling Group, Kubernetes Node/Deployment/HPA, Serverless Function.
  - *流量接入与网络*: DNS, CDN, WAF, 负载均衡器 (L4/L7), Ingress/API 网关, Service Mesh, NAT 网关, 防火墙, 专线/VPN.
  - *应用与中间件*: Web 服务, 后端微服务, BFF, 定时任务, 工作流引擎, 配置中心, 服务注册与发现.
  - *缓存与数据*: Redis (Cache/Cluster/Sentinel), Memcached, 关系型数据库 (MySQL/PostgreSQL 主从/读写分离), NoSQL (MongoDB), 时序数据库, 对象存储 (OSS/S3), 搜索引擎 (Elasticsearch/OpenSearch), 数据仓库.
  - *异步与流式*: 消息队列 (Kafka/RabbitMQ/RocketMQ), 延迟队列, 事件总线, 流处理节点.
  - *可观测与治理*: 日志系统 (ELK/Loki), 指标系统 (Prometheus), 链路追踪 (Jaeger/SkyWalking), 限流器, 熔断器, 降级开关, 灰度发布控制器.
  - *安全与身份*: IAM, KMS, Secret Manager, 证书服务, 审计日志.
- **画布 (Canvas)**: 基于 React Flow 实现，支持组件拖拽、连线（定义依赖/网络流向）、分组（可用区/VPC）.
- **属性配置 (Inspector)**: 配置每个组件的规格（CPU/内存）、策略（重试/超时）、副本数等.

### 2.2 仿真模拟引擎 (Simulation Engine)
- **离散事件模拟**: 不依赖真实 Docker/K8s 资源，而是通过数学模型模拟系统行为，降低运行成本，支持大规模集群模拟。
- **流量发生器**:
  - 支持配置 QPS 曲线（正弦波、脉冲、阶梯增长、突刺、节假日周期）。
  - 模拟不同类型的请求特征（读多写少、计算密集型、长连接、批处理）。
  - 支持多租户流量与流量染色（白名单、灰度流量、机器人流量）。
- **资源消耗模型**:
  - 基于排队论模拟请求处理延迟。
  - CPU/内存/磁盘 IOPS/网络带宽 饱和度计算 -> 触发熔断、限流、降级或崩溃。
  - 组件级容量模型：连接池、线程池、队列深度、缓存命中率、GC 抖动。
- **一致性与数据模型**:
  - 主从复制延迟、读写分离一致性窗口、缓存双写不一致、最终一致性收敛。
  - 搜索索引延迟、消息堆积、消费重平衡、幂等冲突。
- **自动化策略模型**:
  - HPA/ASG 扩缩容规则、冷启动成本、预热策略、抢占式资源回收。
  - 熔断/重试/退避策略与连锁故障传播。
- **仿真事件库 (Event Library)**:
  - *流量事件*: 洪峰突增、突降、热点 Key、慢查询放大、恶意爬虫流量。
  - *基础设施事件*: 可用区故障、网络分区、DNS 解析异常、CDN 回源失败、磁盘打满。
  - *中间件事件*: Redis 淘汰风暴/雪崩、Kafka 分区 Leader 切换、ES 分片失衡。
  - *应用事件*: 发布失败、版本回滚、内存泄漏、线程池耗尽、连接池耗尽。
  - *安全事件*: 证书过期、密钥轮换失败、突发高频鉴权失败、WAF 误杀。
  - *运营事件*: 配置误发布、手工扩容延迟、成本超预算告警触发策略收缩。

### 2.3 场景与演练 (Scenarios)
- **新手引导**: "部署你的第一个高可用 Web 服务"。
- **进阶挑战**: "应对双 11 流量洪峰"（需设计缓存、降级策略）。
- **故障演练**: 随机注入故障（网线被挖断、数据库主从延迟、Pod OOM）。
- **推荐预设场景库**:
  - *单体到微服务迁移*: 引入 API 网关、服务拆分、数据库拆分后的依赖治理。
  - *电商大促秒杀*: CDN + Redis + MQ 削峰填谷，保护核心下单链路。
  - *搜索与推荐平台*: ES 索引延迟、热分片、召回服务降级与兜底策略。
  - *全球化多地域部署*: 多活架构、就近接入、跨地域复制与故障切流。
  - *高监管金融场景*: 全链路审计、密钥轮换、故障下的强一致与限流。
  - *内容平台静态分发*: CDN 回源链路优化、对象存储吞吐瓶颈与带宽成本控制。
  - *实时数据平台*: Kafka 堆积、消费延迟、回压策略与自动扩容。
  - *发布与变更管理*: 金丝雀发布、蓝绿切换、自动回滚与发布窗口管控。
  - *成本优化挑战*: 在 SLA 约束下做实例规格、缓存层级和带宽成本平衡。

### 2.4 监控与反馈 (Dashboard)
- 实时显示 QPS, Latency (P99/P95), Error Rate.
- 资源利用率热力图。
- 架构评分系统（基于成本、可用性、性能打分）。

## 3. 技术栈选型

- **前端框架**: React 18 + TypeScript + Vite
- **UI 组件库**: Tailwind CSS + ShadcnUI (Ant Design)
- **可视化引擎**: React Flow (或 X6) - 处理节点与连线
- **状态管理**: Zustand / Jotai
- **模拟核心**: 纯前端实现的 Agent-based Simulation (便于部署和演示)，后续可扩展为 WebAssembly 或 Node.js 后端。
- **图表库**: Recharts (展示监控数据)

## 4. 实施路线图 (Roadmap)

### 阶段一：MVP 原型 (预计 1-2 周)
- [ ] 初始化项目脚手架。
- [ ] 实现基础画布：支持拖拽 Load Balancer, Web Server, Database 三种核心组件。
- [ ] 实现基础连线：定义流量流向。
- [ ] 简易模拟器：设定固定 QPS，计算各节点负载，简单的红/绿状态反馈。

### 阶段二：核心模拟逻辑 (预计 2 周)
- [ ] 引入时间轴控制（开始、暂停、加速）。
- [ ] 实现请求链路追踪模拟（Request Tracing）。
- [ ] 完善资源模型：CPU/内存限制对响应时间的影响。
- [ ] 添加基础图表：实时 QPS 和 错误率。

### 阶段三：丰富组件与场景 (预计 2 周)
- [ ] 增加 Cache (Redis), Search (Elasticsearch), MQ (Kafka) 与 CDN 组件。
- [ ] 实现"配置面板"：允许调整组件参数（如连接池大小）。
- [ ] 预设至少 6 个典型场景关卡，并支持故障注入模板。

### 阶段四：高级特性
- [ ] 引入"成本"概念，计算架构 ROI。
- [ ] 保存/分享架构设计快照。

```
/src
  /components

    /dashboard    # 监控仪表盘
  /engine         # 核心模拟引擎
    /nodes        # 节点行为定义
    /events       # 事件系统
    /scenarios    # 场景配置
  /store          # 全局状态
```
