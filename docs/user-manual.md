# Deployment Playground 用户使用手册

## 1. 手册目的

本手册面向初次使用者、培训讲师与演练组织者，帮助你快速完成：

- 拓扑搭建
- 仿真运行
- 故障注入与编排
- 指标观测与复盘
- 快照留存与方案对比

## 2. 适用对象

- 架构师：用于方案推演与指标对比
- SRE/运维：用于故障演练与恢复策略验证
- 培训讲师：用于课堂演示与通关考核
- 研发团队：用于发布治理与性能风险预演

## 3. 环境准备

### 3.1 启动项目

```bash
npm ci
npm run dev
```

浏览器访问：`http://localhost:5173/`

### 3.2 术语帮助说明

界面中核心术语后都带有 `?` 图标：

- 点击问号可查看「术语解释 + 常见使用场景」
- 点击术语气泡外部区域会自动关闭

## 4. 界面总览

页面主要分为四个区域：

1. 顶部工具栏：场景选择、开始/暂停、单步仿真、流量曲线、速度、模板操作
2. 左侧组件库：搜索并添加组件节点
3. 中央画布：拓扑建模与链路连接
4. 右侧仿真参数区：节点/观测/事件/复盘四个标签页

## 5. 快速上手（10 分钟通关）

### 第 1 步：选择场景

- 在顶部「场景」下拉中选择可用场景（未解锁场景会标注）
- 阅读场景描述，明确目标指标和约束

### 第 2 步：搭建或调整拓扑

- 从左侧「组件库」点击添加节点
- 在画布中拖动节点位置，按业务链路连接节点
- 若已有模板，使用顶部「模板 -> 应用模板」快速初始化

### 第 3 步：配置节点参数

切到右侧「节点」页签：

- 在 Inspector 中配置：
  - 节点名称
  - 容量
  - 基础延迟
  - 可用区
- 使用「批量编辑」一次修改多个已选节点
- 通过「固定 QPS」滑条设置压测基线

### 第 4 步：运行仿真

- 点击「开始」进行连续仿真
- 点击「单步仿真」逐 Tick 推进，便于课堂讲解和精细调试

### 第 5 步：查看观测结果

切到右侧「观测」页签，重点关注：

- 系统指标：QPS、P95、P99、Error Rate、Availability、Cost
- 评分模型：性能、稳定性、成本、可恢复性、安全性
- 资源热力图：快速定位热点节点
- 调用链路视图：识别关键链路瓶颈
- 性能剖析：评估仿真引擎吞吐与帧率

### 第 6 步：故障演练

切到右侧「事件」页签：

- 选择事件并点击「注入事件」
- 观察激活事件列表与事件时间线
- 使用「事件回放」进行复盘讲解

### 第 7 步：复盘与留档

切到右侧「复盘」页签：

- 查看场景进度与通关状态
- 保存架构快照
- 对比两版方案指标差异
- 阅读自动复盘报告（根因、改进项、推荐架构调整）

## 6. 四大核心页签详细说明

## 6.1 节点

用途：修改模型输入参数、观察节点状态。

建议操作顺序：

1. 先调容量、基础延迟
2. 再设固定 QPS
3. 最后看成功率、超时率、队列、限流/降级变化

## 6.2 观测

用途：判断系统是否达标并定位瓶颈。

推荐判读顺序：

1. 先看 Error Rate / Availability 是否越界
2. 再看 P95 / P99 是否超预算
3. 用热力图找热点节点
4. 用链路视图确认瓶颈传播路径
5. 最后参考评分模型做整体结论

## 6.3 事件

用途：模拟故障发生与恢复过程。

### 单次注入

- 适合课堂讲解单点故障

### 事件编排

- 支持串行/并行
- 支持概率触发
- 支持起止 Tick 时间窗
- 支持故障种子，保证可重复演练

## 6.4 复盘

用途：沉淀结果、比较方案、形成行动项。

- 架构快照：保留拓扑 + 参数 + 指标结果
- 方案对比：比较基线与目标方案
- 自动复盘：输出结构化结论，便于汇报

## 7. 模板与快照使用建议

## 7.1 模板

- 官方模板：用于标准教学起点
- 我的模板：用于团队内部复用

推荐做法：

- 每个业务域维护 1~2 个标准模板
- 每次演练前先应用模板再微调

## 7.2 快照

- 快照用于阶段留档与版本回溯
- 支持导出/导入，便于跨人协作

推荐做法：

- 每轮实验至少保存「基线快照」与「优化后快照」
- 快照命名建议包含：场景名 + 日期 + 版本号

## 8. 培训演练建议流程

1. 选择场景并说明通关目标
2. 使用模板加载初始拓扑
3. 运行基线并保存快照
4. 注入单故障与组合故障
5. 调整架构参数/治理策略
6. 对比快照并讲解复盘报告
7. 记录行动项并进入下一轮

## 9. 常见问题（FAQ）

### Q1：开始后指标没有变化？

- 检查是否建立了完整链路（入口节点到下游）
- 检查固定 QPS 是否过低
- 检查是否处于暂停状态

### Q2：场景无法选择？

- 该场景可能尚未解锁，先完成前置场景并达到星级条件

### Q3：为什么同一编排每次结果不同？

- 设置固定「故障种子」
- 不变更拓扑与参数
- 在同一序列下重放

### Q4：如何把结果给团队复盘？

- 在复盘页保存快照
- 导出快照文件
- 导入后进行方案对比与报告讲解

- 命名规范：模板、快照统一使用业务名+日期+版本
