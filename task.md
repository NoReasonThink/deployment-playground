# Deployment Playground 详细开发待办 (Task Breakdown)

## 0. 交付目标
- 将 `plan.md` 中扩展后的组件库、仿真事件库、场景库落地为可迭代的开发任务。
- 先完成可演示 MVP，再逐步提升仿真真实性、可玩性与可观测能力。
- 所有任务按“可单独验收”的粒度拆分，便于并行推进和阶段复盘。

## 1. 基础工程与架构底座
- [x] 初始化前端工程（React 18 + TypeScript + Vite）并统一代码规范。
- [x] 建立目录结构（components/engine/store/scenarios/dashboard）与模块边界。
- [ ] 统一 UI 设计系统（Tailwind + ShadcnUI）及主题变量。
- [ ] 设计全局状态分层（画布状态、仿真状态、监控状态、场景状态）。
- [ ] 建立领域模型类型定义（节点、连线、事件、指标、评分、场景）。
- [ ] 建立本地持久化机制（草稿自动保存、恢复最近一次编辑）。
- [ ] 建立错误边界与全局异常上报机制。

## 2. 画布与可视化设计器
- [x] 实现基础画布能力（拖拽、缩放、对齐、吸附、框选、分组）。
- [x] 实现节点连接规则引擎（可连接矩阵、环路检测、非法链路提示）。
- [x] 建立组件库分组与搜索（按“网络/应用/缓存/数据/治理/安全”分类）。
- [x] 实现首批节点组件（LB、API 网关、Web、Service、DB、Redis、MQ、CDN、ES）。
- [x] 实现高级节点组件（Service Mesh、对象存储、监控、追踪、限流/熔断器、WAF、IAM）。
- [x] 实现 Inspector 配置面板（规格、策略、副本、依赖、SLA 参数）。
- [x] 实现批量编辑能力（多选节点统一修改副本、资源上限、超时策略）。
- [x] 实现模板化拓扑生成功能（单体、三层、微服务、多可用区）。

## 3. 仿真内核与事件系统
- [x] 设计离散事件调度器（事件队列、优先级、时间推进、暂停/恢复）。
- [x] 实现流量发生器（正弦、脉冲、阶梯、突刺、周期流量）。
- [x] 实现请求生命周期模型（入口、路由、重试、超时、回退、失败）。
- [x] 实现资源消耗模型（CPU、内存、带宽、IOPS、队列深度、连接池）。
- [x] 实现缓存模型（命中率、淘汰策略、穿透/击穿/雪崩）。
- [x] 实现数据库模型（主从延迟、读写分离、一致性窗口）。
- [x] 实现消息系统模型（生产速率、消费速率、堆积、重平衡）。
- [x] 实现搜索系统模型（索引延迟、分片负载、查询退化）。
- [x] 实现自动扩缩容模型（HPA/ASG 规则、冷启动、预热）。
- [x] 构建策略执行器（限流、熔断、降级、重试退避、故障转移）。

## 4. 故障注入与演练事件库
- [x] 建立事件元模型（触发条件、影响范围、持续时间、恢复方式）。
- [x] 实现流量类事件（热点 Key、突发流量、恶意请求）。
- [x] 实现网络类事件（网络分区、DNS 异常、CDN 回源失败）。
- [x] 实现中间件事件（Redis 淘汰风暴、Kafka Leader 切换、ES 热分片）。
- [x] 实现应用层事件（版本发布失败、线程池耗尽、连接池耗尽、内存泄漏）。
- [x] 实现安全类事件（证书过期、密钥轮换失败、鉴权故障、WAF 误杀）。
- [x] 实现运营类事件（配置误发布、人工扩容延迟、预算超限触发策略降级）。
- [x] 支持事件编排（串行/并行注入、概率触发、时间窗触发）。

## 5. 场景系统与关卡设计
- [x] 搭建场景 DSL（目标、初始拓扑、限制条件、评分权重、通关条件）。
- [x] 新手场景：部署高可用 Web 服务并通过基础 SLA 指标。
- [x] 电商大促场景：CDN + Redis + MQ 抗洪峰，保护核心下单链路。
- [x] 搜索推荐场景：处理 ES 索引延迟与热分片导致的高延迟。
- [x] 多地域容灾场景：实现跨地域流量切换与数据复制容错。
- [x] 金融合规场景：兼顾审计、密钥管理与高可用约束。
- [x] 成本优化场景：在 SLA 不退化前提下降低资源成本。
- [x] 发布治理场景：金丝雀发布失败后自动回滚并恢复指标。
- [x] 实现场景进度系统（解锁、评分星级、复盘建议）。

## 6. 监控仪表盘与评分体系
- [x] 实现核心指标面板（QPS、P95/P99、Error Rate、可用性、成本）。
- [x] 实现资源热力图（节点 CPU/内存/带宽/队列负载）。
- [x] 实现调用链路视图（关键链路时延分解与瓶颈高亮）。
- [x] 实现事件时间线（注入事件、告警、扩容、回滚）。
- [x] 设计评分模型（性能、稳定性、成本、可恢复性、安全性）。
- [x] 输出自动复盘报告（根因、改进项、推荐架构调整）。

## 7. 保存分享与协作能力
- [x] 实现架构快照保存（拓扑、参数、场景、结果）。
- [x] 实现方案分享（只读链接或导出文件）。
- [x] 实现方案对比（两版拓扑指标差异与成本差异）。
- [x] 实现模板市场基础能力（官方模板、我的模板）。

## 8. 质量保障与工程化
- [x] 建立单元测试（模型计算、事件引擎、评分器）。

- [x] 建立端到端测试（典型场景通关流程）。
- [x] 建立性能基线（大规模节点/事件下帧率与仿真吞吐）。
- [x] 建立可观测调试工具（内部状态快照、事件回放、性能剖析）。
- [x] 建立 CI 流水线（检查、测试、构建、产物归档）。

- [x] 验收标准：可完成从建模到运行到复盘的完整闭环。

- [ ] 验收标准：支持可重复故障注入并稳定复现瓶颈与恢复过程。

## 10. 依赖与风险清单
- [ ] 明确仿真精度边界，避免“过度拟真”导致实现复杂度失控。
- [ ] 明确节点参数默认值与上下限，降低用户配置门槛。
